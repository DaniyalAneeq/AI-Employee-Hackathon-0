## LinkedIn Post to Publish

**Topic:** Building a Personal AI Employee with Claude Code

## Post Content

I just built something I have been thinking about for months.

An AI Employee that runs 24/7, reads my emails, monitors my LinkedIn, and drafts replies for my approval.

Not a chatbot. Not a copilot. An actual autonomous agent with memory, rules, and a human-in-the-loop safety layer.

Here is the stack I used:
- Claude Code as the reasoning engine
- Obsidian as the vault and dashboard
- Playwright for browser automation
- Gmail API for email monitoring
- Python daemons as the sensory layer

The key insight: the bottleneck was never the AI. It was giving the AI a structured way to act without going rogue.

The solution was simple — file-based approvals. Every sensitive action (email reply, LinkedIn post, payment) writes an approval file. The AI waits. I review. I move the file. The action executes.

Human in the loop, but zero friction.

This is what I built for the GIAIC Hackathon 0. And honestly, it already saves me hours every week.

If you are building something similar or curious how it works, drop a comment or DM me.
